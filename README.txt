선행 계산기의 경우

- 버튼의 배치 및 Connect 에, QVector 템플릿으로 생성된 QPushButton 의 선형 자료구조를 활용했던 바 있다.

- 버튼의 배치는 그냥 하나씩 위치좌표 줘서..


후행 계산기의 경우

- 배치 역할은 GridLayout 에 맡긴다. 이는 똑같이 16버튼에 대한 반복문이 처리한다.

- 반면 동작 역할은 ButtonGroup 에 맡긴다. addButton 메소드를 활용하면, id 와 함께 각 버튼들을 한 그룹에 묶어놓을 수 있는데, 이는 동작의 객체지향 및 일관성상에서 궁금증이 좀 생깁니다.

- 위 두 조치 덕분에, 후행 계산기에서는 QObject::connect 함수에 버튼그룹클래스의 idClicked(int) 를 활용해, 한번에 모든 버튼들과 SLOT 을 연결해줄 수 있다.
ㄴ> SLOT 은, idClicked 가 던져주는 int id 값을 가지고 각 버튼을 구분해서 버튼그룹으로부터 불러와 조건분기하게 된다
